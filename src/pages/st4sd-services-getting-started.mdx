**ST4SD Services** is a collection of microservices that power the ST4SD technologies.
The APIs of these microservices act as a bridge between users and ST4SD technologies.
Any researcher comfortable with programming can start using ST4SD Services.

## Why ST4SD Services ?

<div className="expressive-content-list-group">
  <ExpressiveList
    title="APIs to interact with ST4SD technologies"
    background="true"
    pictogram={<AppDeveloper />}>

Use the APIs to create, parameterise, run, and monitor the progress of experiments.

  </ExpressiveList>


  <ExpressiveList
    title="Metadata database"
    background="true"
    pictogram={<Construct />}>

Executing an experiment generates metadata, intermediate files, and outputs.
ST4SD automatically maintains a record of this information in its Datastore.

  </ExpressiveList>

  <ExpressiveList
    title="Memoization"
    background="true"
    pictogram={<AcceleratedComputing />}>

Use memoization to reuse results of previous calculations rather than executing them again.
This can significantly improve the execution time of your experiments.

  </ExpressiveList>
</div>


## Try it yourself

Use [**ST4SD Cloud**](/st4sd-cloud-getting-started#try-it-yourself) to deploy the services on OpenShift.
Next, install the [**ST4SD Core**](/st4sd-core-getting-started#try-it-yourself) python client code.

```
pip install st4sd-runtime-core
```

Finally, use the python client code to try out some of our example Jupyter [notebooks](https://github.com/st4sd/st4sd-examples/tree/main/notebooks):

- [Create, run, and monitor experiments](https://github.com/st4sd/st4sd-examples/blob/main/notebooks/ST4SD%20Runtime%20API%20Example.ipynb)
- [Query the Datastore for metadata, files, and logs that experiments generate](https://github.com/st4sd/st4sd-examples/blob/main/notebooks/ST4SD%20Datastore%20-%20Common%20Query%20Examples.ipynb)
